# apps/web/components/inventory-management-view.tsx

- **役割**: 在庫KPI、在庫推移、在庫アラート、全商品在庫の概要
- **KPI/チャート**: 色・指標の一貫性を維持。前処理はサーバ側、描画はクライアント。

- **セキュリティ**: 機密（GAS_API_KEY等）はサーバのみ参照。クライアントへ出さない。
- **キャッシュ**: GAS=120秒、UI=ISR+SWR の多層キャッシュを意識する。

## データマッピング
- 在庫切れ/不足/過剰数: 「商品状態」の `現在在庫数` を閾値分類（=0, <20, >150）
- 在庫金額: 各商品の `現在在庫数 * 平均販売単価`（平均販売単価は「商品別売上集計」の `実質売上/販売数量`）の合計（簡易）
- 在庫推移: 「全体在庫日次集計」の `在庫数`
- 在庫アラート一覧: 「商品状態」から `現在在庫数` と `推奨発注数`、商品名は「商品マスタ」JOIN
- 全商品在庫一覧: 「商品状態」の `現在在庫数` と「商品マスタ」の `商品名`

## 必要API
- GET /api/gas/alerts
- GET /api/gas/dashboard?from&to&grain
- GET /api/gas/products?from&to&grain
